# Amazon Deal Watcher (бесплатная версия)

Отслеживает скидки на Amazon по вашим брендам, размерам и порогу скидки —
**парся страницы поиска Amazon напрямую** (без платных API) — и присылает
уведомление в **Telegram**, когда появляется что-то подходящее. Разворачивается
бесплатно на **Render.com**.

Текущие правила (меняются в `config.yaml`):

- **Обувь** — HUGO и GANT, размер 10 US, скидка **от 60%**
- **Одежда** — GANT, Lacoste, Hugo Boss, размеры M и L, скидка **от 70%**
- Маркетплейс: Amazon.com (US), проверка каждые 20 минут

> ⚠️ **Честное предупреждение про капчу.** Amazon блокирует автоматические
> запросы с серверных IP (дата-центров Render) и показывает капчу. Скрипт это
> распознаёт и пишет в лог, но обойти не может. Поэтому бесплатный парсинг с
> сервера **может работать нестабильно или перестать работать**. Это осознанный
> компромисс ради полностью бесплатного решения. Если нужна надёжность —
> вернитесь к версии на Keepa API (~€19/мес) или запускайте локально из своего
> браузера.

## Что понадобится (всё бесплатно)

1. **Telegram-бот** и ваш chat_id
2. Аккаунт **GitHub** — для деплоя
3. Аккаунт **Render.com**
4. Аккаунт **UptimeRobot** — чтобы Render не засыпал

---

## Шаг 1. Telegram-бот

1. В Telegram напишите **@BotFather** → `/newbot` → задайте имя и username.
   Получите токен вида `123456789:AА...` — это `TELEGRAM_BOT_TOKEN`.
2. Найдите своего бота и нажмите **Start** (обязательно).
3. Напишите **@userinfobot** — он пришлёт ваш числовой ID — это `TELEGRAM_CHAT_ID`.

---

## Шаг 2. Код на GitHub

Загрузите файлы этой папки в репозиторий GitHub (файл `.env` не загружайте).

---

## Шаг 3. Деплой на Render

1. https://render.com → **New → Web Service**.
2. Источник — ваш репозиторий (или вкладка **Public Git Repository** + ссылка).
3. Instance Type: **Free**. Runtime подхватится из Dockerfile.
4. В **Environment** добавьте переменные:
   - `TELEGRAM_BOT_TOKEN` = токен бота
   - `TELEGRAM_CHAT_ID` = ваш chat id
   - `RUN_MODE` = `loop`
   - `CHECK_INTERVAL_MINUTES` = `20`
5. **Create Web Service**, дождитесь **Live**, скопируйте адрес сервиса.

---

## Шаг 4. Keep-alive (UptimeRobot)

Free-сервис Render засыпает после 15 мин простоя. Чтобы цикл не вставал:

1. https://uptimerobot.com → **Add New Monitor**.
2. Type: **HTTP(s)**, URL: адрес сервиса Render, Interval: **5 минут**.
3. Save.

Готово — сервис работает и шлёт уведомления в Telegram при подходящих скидках.

---

## Как менять правила

Правьте `config.yaml` в репозитории: `query` (поисковый запрос), `sizes`,
`min_discount`. Render с `autoDeploy` пересоберётся сам (или нажмите Manual
Deploy). Частоту можно менять переменной `CHECK_INTERVAL_MINUTES`.

---

## Локальный запуск

```bash
cp .env.example .env      # впишите токен и chat_id
pip install -r requirements.txt
set -a; source .env; set +a
RUN_MODE=once python watcher.py
```

> Локально с домашнего IP парсинг обычно работает лучше, чем с сервера —
> Amazon реже блокирует бытовые IP.

---

## Ограничения (честно)

- **Капча/блокировка.** Главный риск, см. предупреждение выше.
- **Размеры — best-effort.** Проверяются по названию товара; иногда стоит
  перепроверить на странице (ссылка есть в уведомлении).
- **Фейковые скидки.** % считается от зачёркнутой цены Amazon; она бывает
  завышенной. Полностью отсеять ложные срабатывания без истории цен нельзя.
- **Дедуп** хранится в `seen_deals.json`; на free Render может обнуляться при
  перезапуске — тогда возможен один повторный алерт.
