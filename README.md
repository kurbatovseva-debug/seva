# Amazon Deal Watcher

Отслеживает скидки на Amazon по вашим брендам, размерам и порогу скидки через
**Keepa API** и присылает уведомление в **Telegram**, когда появляется что-то
подходящее. Работает 24/7 на сервере (инструкция — под бесплатный **Render.com**).

Текущие правила (меняются в `config.yaml`):

- **Обувь** — бренды HUGO и GANT, размер 10 US, скидка **от 60%**
- **Одежда** — GANT, Lacoste, Hugo Boss, размеры M и L, скидка **от 70%**
- Маркетплейс: Amazon.com (US), проверка каждые 15 минут

---

## Что вам понадобится (по порядку)

1. Ключ **Keepa API** (платно, ~€19/мес)
2. **Telegram-бот** и ваш chat_id (бесплатно)
3. Аккаунт **GitHub** (бесплатно) — для деплоя на Render
4. Аккаунт **Render.com** (бесплатно)
5. Аккаунт **UptimeRobot** (бесплатно) — чтобы Render не засыпал

Ниже — каждый шаг подробно.

---

## Шаг 1. Keepa API ключ

1. Зарегистрируйтесь на https://keepa.com
2. Оформите подписку и API-доступ: https://keepa.com/#!api
   (базовый тариф даёт 1 токен/мин — этого хватает для нескольких брендов).
3. Скопируйте **API key** из личного кабинета. Это `KEEPA_API_KEY`.

> Зачем Keepa: скрипт с сервера нельзя надёжно «парсить» Amazon напрямую —
> его быстро блокируют капчей. Keepa отдаёт цены, историю и скидки легально
> через API.

---

## Шаг 2. Telegram-бот

1. В Telegram напишите боту **@BotFather** → команда `/newbot` → задайте имя.
   Он выдаст токен вида `123456789:AА...` — это `TELEGRAM_BOT_TOKEN`.
2. Найдите вашего нового бота в поиске и нажмите **Start** (обязательно, иначе
   бот не сможет вам писать).
3. Узнайте свой `TELEGRAM_CHAT_ID`: напишите боту **@userinfobot** — он пришлёт
   ваш числовой ID. Это `TELEGRAM_CHAT_ID`.

---

## Шаг 3. Загрузите проект на GitHub

1. Создайте новый **приватный** репозиторий на https://github.com/new
2. Загрузите в него все файлы из этой папки (кнопкой «uploading an existing
   file» или через git). **Файл `.env` не загружайте** — он в `.gitignore`.

---

## Шаг 4. Деплой на Render (бесплатно)

1. Зайдите на https://render.com через GitHub-аккаунт.
2. **New → Web Service** → выберите ваш репозиторий.
3. Render прочитает `render.yaml` автоматически. Если попросит вручную:
   - Runtime: **Python**
   - Build Command: `pip install -r requirements.txt`
   - Start Command: `python watcher.py`
   - Instance Type: **Free**
4. В разделе **Environment** добавьте переменные (Add Environment Variable):
   - `KEEPA_API_KEY` = ваш ключ
   - `TELEGRAM_BOT_TOKEN` = токен бота
   - `TELEGRAM_CHAT_ID` = ваш chat id
   - `RUN_MODE` = `loop`
   - `CHECK_INTERVAL_MINUTES` = `15`
5. **Create Web Service**. Дождитесь статуса **Live** и скопируйте адрес
   сервиса (вида `https://amazon-deal-watcher.onrender.com`).

---

## Шаг 5. Keep-alive (чтобы Render не засыпал)

Бесплатный Render-сервис засыпает после 15 минут простоя. Чтобы цикл проверок
не вставал, настройте бесплатный пинг:

1. Зарегистрируйтесь на https://uptimerobot.com
2. **Add New Monitor**:
   - Monitor Type: **HTTP(s)**
   - URL: адрес вашего сервиса с Render (из шага 4)
   - Monitoring Interval: **5 минут**
3. Save. Теперь UptimeRobot стучится к сервису и держит его «бодрым».

> Альтернатива UptimeRobot — https://cron-job.org (тоже бесплатно).

Готово. Сервис работает 24/7 и шлёт уведомления в Telegram, когда находит
подходящую скидку.

---

## Как менять правила

Откройте `config.yaml` в репозитории GitHub, отредактируйте бренды/размеры/
пороги и сохраните — Render с `autoDeploy: true` пересоберёт сервис сам.
Примеры правок есть в комментариях внутри файла.

Быстрые переключатели через Environment на Render:
- `CHECK_INTERVAL_MINUTES` — частота проверок.
- `RUN_MODE=once` — один прогон и выход (для внешнего cron вместо цикла).

## Локальный запуск (для проверки перед деплоем)

```bash
cp .env.example .env      # впишите свои ключи
pip install -r requirements.txt
set -a; source .env; set +a
RUN_MODE=once python watcher.py    # один тестовый прогон
```

Или через Docker:

```bash
docker build -t deal-watcher .
docker run --env-file .env deal-watcher
```

## Важные ограничения (честно)

- **Размеры — best-effort.** Keepa надёжно отдаёт цены и скидки, но не всегда
  точно — «есть ли ваш размер прямо сейчас в наличии». Скрипт проверяет размер
  по названию и атрибутам вариаций; иногда стоит перепроверить на самой странице
  (ссылка есть в уведомлении).
- **Фейковые скидки.** % считается от list price и 90-дневного среднего (берётся
  большее), чтобы отсеять «надутые» старые цены. Это снижает, но не исключает
  ложные срабатывания.
- **Free Render.** Если пингер сбоит — сервис засыпает и проверки встают до
  следующего пробуждения. Для железобетонного 24/7 возьмите платный Render
  Background Worker ($7/мес) или VPS (Hetzner CX23 ~€5.49/мес) — тогда keep-alive
  и health-эндпоинт не нужны, просто `RUN_MODE=loop`.
- **Дедуп.** Повторно об одном и том же предложении по той же цене не уведомляем
  (хранится в `seen_deals.json`). На free Render этот файл может обнуляться при
  перезапуске — тогда возможен один повторный алерт, это не критично.
